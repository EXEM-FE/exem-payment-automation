# 06. 보안 / 프라이버시

## 위협 모델 (간단)

| 위협 | 가능성 | 대응 |
|---|---|---|
| 공개 인터넷에서 API 접근 | 있음 | **PIN 시크릿** + 1회용·1h + 24h TTL |
| 다른 사용자가 흉내 | 부서+이름만으론 가능 | **PIN 시크릿** + 1회용·1h |
| 서버 침해 / 메모리 덤프 | 낮음 | 카드 정보 0, 사진은 24h TTL |
| 디바이스 분실 (폰/PC) | 있음 | 로컬 데이터는 자기 책임. "모든 데이터 삭제" 버튼 명확 노출 |
| 클립보드/스크린샷 통한 PIN 누출 | 낮음 | PIN 1회용·1h |
| 회사가 직원 데이터 들여다본다는 인상 | **가장 큰 위험** | "메모리만, 카드정보 0, 24h, 폐기 즉시" 가시화 |

## 첫 번째 가드: 데이터 분리

서버에 **올라오지 않는** 정보:
- 카드번호 (마스킹조차도 보내지 않음)
- 사업자번호
- 승인번호
- 실제 가맹점 정식명
- 실제 청구금액

이것들은 **명세서 .xls**에만 존재하며, PC 브라우저 메모리에서만 처리되고 즉시 휘발됨.

서버가 보는 정보는 사실상:
- 일자, 사용자가 적은 가게 메모, 예상 금액
- 함께 식사한 사람 이름
- 업무 내용("야근 식대 4인")
- 영수증 사진

## 두 번째 가드: 메모리 + TTL

- DB 없음 → 디스크 흔적 0
- 사용자별 슬롯 24h TTL
- PIN 1회용 + 1h TTL
- 다운로드 완료 시 즉시 DELETE
- 서버 재시작 = 모든 데이터 청소 (의도된 동작)

## 세 번째 가드: 인증

| 방법 | 1차 | 2차+ |
|---|---|---|
| 부서 + 이름 | ✅ 식별자 | (변경 없음) |
| 일회용 PIN | ✅ 시크릿 | (변경 없음) |
| 사번 | ❌ | ✅ 추가 식별자 |
| 다우오피스 SSO | ❌ | ✅ 검토 |

> 1차는 **부서+이름+PIN**으로만 시작. 동일 부서·동일 이름 충돌이 없는 환경 전제.
> 2차에서 사번/SSO를 검토.

## 네 번째 가드: 공개 배포 + HTTPS

- Vercel 공개 배포를 전제로 한다
- HTTPS는 Vercel 배포에서 기본 제공
- 접근 통제는 네트워크 제한 대신 PIN·TTL·데이터 분리 원칙에 둔다

## 사용자에게 보여줄 것 (신뢰 시그널)

화면 곳곳에서 명시:
- "카드정보는 서버를 거치지 않습니다 (명세서는 PC에서만)"
- "서버 데이터는 24시간 후 자동 삭제됩니다"
- "다운로드 완료 시 서버 데이터는 즉시 삭제됩니다"
- "이 서버는 카드번호·가맹점·청구금액을 보지 않습니다"

특히 **첫 푸시 직전 모달**에서 한 번 명확히 고지.

## 디바이스 데이터

| 저장소 | 용도 | 사용자 통제 |
|---|---|---|
| localStorage | 부서·이름 (식별자) | 설정 > 수정/초기화 |
| IndexedDB (모바일) | 저널, 사진, 멤버 자동완성 | 설정 > 이번 달 초기화 / 모든 데이터 삭제 |
| IndexedDB (PC) | 임시 작업 (pull한 데이터 + 명세서 파싱) | 다운로드 완료 시 자동 청소 |

## 카메라 / 사진 권한

- `<input type="file" accept="image/*" capture="environment">`
- 별도 권한 다이얼로그 X (브라우저 표준 file input)
- 한번 찍은 사진은 IndexedDB에 즉시 저장, 카메라 롤에는 자동 저장 안 함 (명시 안 함)

## 카드번호 마스킹 (혹시 들어온다면)

명세서를 PC에서 파싱해 .xlsx에 채울 때, 모든 카드번호는 마스킹 형식 그대로 유지: `****-3910-****-3811`. 마스킹 해제 처리는 어디에도 없음.

## 비상 절차

| 상황 | 동작 |
|---|---|
| 서버 침해 의심 | 서버 재시작 → 모든 메모리 청소. 인프라팀 조사. |
| 사용자가 "내 데이터 다 지워줘" 요청 | 설정 > "모든 데이터 삭제" 안내 + 서버는 `DELETE /api/me` 호출 |
| PIN 분실 | 모바일에서 "새 PIN 발급" |
| 24h 후 PC pull 시도 | 슬롯 만료 → 모바일에서 다시 push |

## 명시적 비목표

- 민감 데이터에 대한 **암호화 저장** — 메모리만 쓰므로 적용 대상 아님
- **감사 로그** — 본문 로깅 안 함 (메타만)
- **SSO/MFA** — 1차 스코프 외
- **개별 영수증 OCR/AI 처리** — 외부 API 콜 0
