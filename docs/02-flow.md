# 02. 사용자 흐름

## 전체 흐름 (한 달 사이클)

```mermaid
sequenceDiagram
    participant M as 📱 모바일 PWA
    participant H as 🌐 Vercel API 허브 (메모리)
    participant P as 💻 PC 정산 콘솔

    Note over M: 일상 — 네트워크 무관, 오프라인 OK
    M->>M: 식사 자리에서 항목 추가 (IndexedDB)
    M->>M: 편의점에서 영수증 사진만 (IndexedDB)

    Note over M: 월말 — 사용자가 직접 트리거
    M->>M: "PC로 보내기" 버튼
    M->>H: POST /push (저널만, 카드정보 X)
    H-->>M: PIN 1234 (1h 만료)
    M->>M: 화면에 PIN 표시

    Note over P: PC 접속
    P->>P: 부서 + 이름 + PIN 입력
    P->>H: POST /pull
    H-->>P: 저널 데이터
    H->>H: PIN 즉시 폐기 (24h TTL은 데이터에만)
    P->>P: 명세서 .xls 업로드 (브라우저 메모리만)
    P->>P: 자동 매칭 + 검토
    P->>P: .xlsx 다운로드 (브라우저 메모리만)
    P->>H: DELETE /me (서버 데이터 즉시 삭제)

    Note over M: 모바일 IndexedDB는 그대로 유지<br/>(다음 달 멤버 자동완성용 빈도 데이터)
```

## 트리거 원칙: 자동 동기화 X, 사용자 명시적 푸시 1회

| 상황 | 동작 |
|---|---|
| 모바일 입력 시 | IndexedDB만 저장 |
| 네트워크 연결 변화 | 자동 동기화 없음 |
| 사용자 "PC로 보내기" 클릭 | 1회 push, PIN 발급 |
| 사용자가 모바일에서 추가 입력 후 다시 "PC로 보내기" | 새 PIN 발급, 이전 PIN 폐기 |
| PC pull 성공 | PIN 즉시 폐기, 데이터는 24h TTL 유지 |
| PC "다운로드 완료" | DELETE 호출, 서버 데이터 즉시 삭제 |

## 일상 시나리오 — 야근 식대 입력 (30초)

```mermaid
flowchart TD
    A[저녁 7시: 사무실에서 야근식대 결제] --> B[폰에서 PWA 실행<br/>홈에 추가돼 앱처럼]
    B --> C[FAB ＋ 탭]
    C --> D[📷 카메라로 영수증 촬영]
    D --> E[금액·가게이름 자동 OCR 안 함<br/>사용자가 직접 빠르게 입력]
    E --> F[멤버 칩 선택<br/>최근 멤버 묶음 1탭]
    F --> G[업무 템플릿: '야근 식대 N인']
    G --> H[저장 → 저널 카드로 표시]
```

소요시간 목표: **30초 이내**.
빠른 입력의 핵심은 ① 카메라 즉시 호출 ② 멤버 칩 자동완성 ③ 업무 템플릿 ④ 계정 자동 추천.

## 일상 시나리오 — 편의점 영수증만 (10초)

```mermaid
flowchart TD
    A[편의점에서 사무용 종이 구매] --> B[폰에서 PWA 실행]
    B --> C[FAB ＋ 탭]
    C --> D[📷 영수증 촬영]
    D --> E[임시저장 클릭<br/>나머지 비워둠]
    E --> F[저널에 ⚠ 표시로 추가됨<br/>나중에 보강 필요]
```

소요시간 목표: **10초**.
멤버·금액·계정 입력은 나중에 미룸. 사진만 잡아두는 게 가치.

## 월말 시나리오 — PC 정산 (5~7분)

```mermaid
flowchart TD
    A[PC 브라우저로 PWA 접속] --> B[부서·이름·PIN 입력]
    B --> C[모바일 데이터 가져오기]
    C --> D[누락된 항목 보강<br/>저널에 ⚠ 있는 것들]
    D --> E[현대카드 MY COMPANY에서<br/>명세서 .xls 다운로드]
    E --> F[명세서 업로드]
    F --> G[자동 매칭 결과 검토<br/>✅⚠❌]
    G --> H{매칭 안 된 행 있음?}
    H -- 명세서에만 있음 --> I[PC에서 인라인 추가 입력]
    H -- 저널에만 있음 --> J[개인사용 처리 또는 삭제]
    H -- 모두 매칭 --> K[미리보기]
    I --> K
    J --> K
    K --> L[.xlsx 다운로드]
    L --> M[다우오피스에서 기안 작성<br/>다운로드한 파일 첨부]
    L --> N[자동: 서버 데이터 삭제]
```

## 오프라인/네트워크 오류 시 UX

입력은 항상 가능. PC로 보내기에서 API 요청이 실패하면 일반 전송 실패 안내만 표시한다.

```
┌─────────────────────────────────┐
│ 📤 PC로 보내기                  │
│                                 │
│ ⚠ 보내지 못했어요               │
│   잠시 후 다시 시도해주세요      │
│                                 │
│ [취소]                          │
└─────────────────────────────────┘
```

## 핵심 비기능 요건

- **오프라인 입력**: 사진 촬영·항목 추가는 네트워크 무관
- **새로고침 무손실**: 모든 입력은 IndexedDB 즉시 저장
- **다운로드 후 폐기**: 서버 데이터, 명세서 임시 데이터 모두 즉시 청소
- **모바일 IndexedDB 보존**: 다음 달 멤버 자동완성·통계용. 사용자가 명시적으로 "초기화"할 때만 삭제.
