# 08. 룰 엔진

## 왜 분리하나

엑셈 경비 규정은 매년 또는 분기별로 갱신된다(2025년판 기준 14차례 부칙 존재). 룰을 코드에 박으면 매번 빌드·배포가 필요하므로 **JSON 설정 파일로 분리**하고, 앱은 룰 파일의 버전을 표시해 사용자가 "내가 보는 룰이 맞나?" 검증할 수 있게 한다.

## 룰 카테고리 (4축)

| 축 | 의미 | 출처 |
|---|---|---|
| **A. Auto** | 명세서 데이터만으로 자동 처리 | 가맹점 패턴, 직급, 1km당 단가 |
| **B. Assist** | 사용자 1탭으로 분기 결정 | 식음료 가맹점에 노출되는 4지선다 |
| **C. Validate** | 룰 위반·누락 경고 | 한도 초과, 영수증 누락, 사전 승인 필요 |
| **D. Notify** | 마감일·소멸 알림 | 매월 7영업일 등 |

## 식음료 4지선다 (Assist)

가맹점 메모가 `food_merchant_patterns`에 매칭되면 자동 노출:

```
[가맹점: 스타벅스 마곡]
이 지출의 성격은?
 ◯ 야근 식대 (20시 이후)        → 복리후생비 + 12,000원/1인 검증
 ◯ 휴일 출근 식대                → 복리후생비 + 15,000원/1인 검증
 ◯ 팀 회식                       → 회식비 + 월 50,000원/1인 누적 검증
 ◯ 고객 미팅 (9~18시, 주류 X)    → 회의비
 ◯ 고객 미팅 (그 외)             → 접대비
```

선택 시 **계정 자동 채움 + 인원 입력 강제**.

## 검증 룰 (Validate)

PC 정산 Step 4 다운로드 직전에 일괄 검증.

| 룰 | 트리거 | 처리 |
|---|---|---|
| 영수증 필수 미첨부 | 가맹점이 receipt_required_patterns 매칭 + 사진 0장 | 다운로드 비활성화 |
| 동반인 미입력 | 식대성 계정 + 사용자명(all) 비어있음 | 다운로드 비활성화 |
| 50만원 접대성 | 단일 행 청구금액 ≥ 500,000 + 계정=접대비 | "별도 지출결의서 필요" 안내 |
| 회식비 월 한도 | (회식비 누적) / 인원 ≥ 50,000 | 경고만 (1차에서는 누적 계산 X) |
| 숙박 80,000 초과 | 계정=출장비 + 단일 행 ≥ 80,000 | "본부장+재무회계 사전 승인" 안내 |
| KTX 특실/항공 | 가맹점=코레일/항공사 + 금액 패턴 | "본부장 승인" 안내 |
| 카드 결제 불가 항목 | 가맹점=주차/통행료 | "현금 사용해야 정상" 경고 (역방향) |

## 룰 갱신 절차

1. 재무회계팀에서 새 규정 PDF 공유받음
2. `rules/YYYY-MM-DD.json` 새 파일 작성
3. PR로 변경분 리뷰 (diff 명확)
4. 빌드 산출물에 포함, 사용자 화면에 새 버전 표시
5. 사용자가 "이전 버전과 비교" 메뉴로 변경점 확인 가능 (2차)

## 2차 보강 — 휴리스틱 매칭 (PC 정산)

가맹점 표기 차이로 인한 매칭 실패를 줄이기 위해 정규화 + Sørensen-Dice bigram 유사도 적용.

```
normalizeMerchant("(주)이니시스 - ㈜우아한형제들") → "이니시스우아한형제들"
normalizeMerchant("우아한형제들") → "우아한형제들"
유사도 계산 → substring 포함, 길이 비율 0.5+ → 매칭 OK
```

매칭 우선순위 (`buildMatches`):
1. 날짜 + 금액 일치 → exact
2. 같은 날, 가맹점 유사도 ≥ 0.6 (금액 다름) → review (저널/명세서 금액 차이 안내)
3. 같은 날만 일치 → review
4. 매칭 실패 → **자동 시드 entry로 채움** (명세서 SSoT — 행은 항상 살아있음)

## 검증 톤 일원화 — 모두 warn

1차 MVP에서 영수증 누락/동반인 누락/50만 초과는 다운로드 block이었으나, 사용자 경험을 부드럽게 하기 위해 **모두 warn으로 변경**. 다운로드는 항상 가능하며, DownloadStep에서 안내 패널로 노출.

각 issue type:
- `receipt`: 영수증 필수 가맹점인데 사진 0장
- `participants`: 식대성 카테고리인데 함께한 사람 비어있음
- `amount`: 단일 행 청구금액 ≥ 500,000 (품의서 사전 작성)
- `per_person`: 야근/휴일 식대 인당 한도 초과
