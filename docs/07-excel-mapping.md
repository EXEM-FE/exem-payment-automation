# 07. 엑셀 템플릿 셀 매핑 명세

> 구현 시 가장 자주 참조하게 될 문서. 채워진 신청서(`(카드)제경비신청서_10월_FE1팀_최기환.xls`) 분석 결과를 그대로 정리.

## 입력: 명세서 (`명세서_YYYYMMDD.xls`)

- 시트 1개: `청구내역상세`
- 헤더 1행, 데이터 2행~
- 14컬럼

| 컬럼 (0-base) | 헤더 | 예시 |
|---|---|---|
| 0 | 이용일자 | `2025.10.13` 또는 `2026-01-05` (문자열) |
| 1 | 카드번호 | `****-3910-0209-3811` |
| 2 | 이용자명 | `최*환` |
| 3 | 사원번호 | `A10241104` |
| 4 | 부서명 | `FE1팀` |
| 5 | 국내이용금액 | `48000` |
| 6 | 국내청구금액 | `48000` |
| 7 | 해외현지금액 | `0` |
| 8 | 통화코드 | `-` |
| 9 | 가맹점 | `(주)이니시스 - ㈜우아한형제들` |
| 10 | 가맹점사업자번호 | `220-81-55597` |
| 11 | 승인번호 | `00572725` |
| 12 | 할부개월수 | `0` |
| 13 | 청구회차 | `0` |

## 출력: 신청서 (`(카드)제경비신청서_M월_부서_성명.xlsx`)

- 시트 4개

```
1. 이용내역명세서
1-1. 지출증빙 첨부(쇼핑몰,편의점,마트,문구점 등)
2. 후불교통,하이패스이용명세서
지출증빙 첨부_예시
```

### 시트 1. `1. 이용내역명세서`

#### 헤더 (1행, 0-base)

| 컬럼 | 헤더 | 비고 |
|---|---|---|
| 0~13 | 명세서 14컬럼 그대로 | 명세서 → 신청서 직접 복사 |
| 14 | (라벨 없음, 합계 셀) | 행별 합계 (=국내청구금액으로 추정) |
| 15 | 계정 | 사용자 입력 |
| 16 | (외근시 기재)\n거래처명/소재지 | 사용자 입력 (외근시만) |
| 17 | 사용자명(all) | 사용자 입력 (참가자 전원) |
| 18 | 업무 상세내용\n(주유는 '거리(km)*이용일수' 함께 기재) | 사용자 입력 |
| 19 | 경비신청금액 | 사용자 입력 (회사 부담) |
| 20 | 개인사용금액 | 자동 계산 (= 청구 - 신청) |

#### 헤더 영역 0행 (헤더 위 안내·합계)

| 위치 | 값 |
|---|---|
| (0, 8) | `<이용내역명세서 붙여넣기 영역>` (라벨) |
| (0, 9) | 합계 (= 국내청구금액 컬럼 SUM) |
| (0, 17) | `<사용자 추가입력 영역>` (라벨) |
| (0, 19) | 합계 (= 경비신청금액 컬럼 SUM) |
| (0, 20) | 합계 (= 개인사용금액 컬럼 SUM) |
| (0, 21) | `1` (의미 미상, 그대로 유지) |

#### 데이터 행 (2행~, 한 명세서 행 = 한 출력 행)

| 컬럼 | 값 |
|---|---|
| 0 | 명세서.이용일자 원문 텍스트 보존 (`2025.10.13`, `2026-01-05` 등) |
| 1~13 | 명세서 그대로 (단, 사업자번호·승인번호는 숫자로 변환되기도 함) |
| 14 | 행 합계 (= col 6 국내청구금액) |
| 15~18 | 사용자 입력 |
| 19 | 사용자 입력 (기본값 = 청구금액) |
| 20 | 자동 = 청구 - 신청 |

#### 매핑 예 (실제 데이터)

명세서 1행 `2025.10.13 / 우아한형제들 / 48000` →
신청서 행 2:
```
(2,0)=45943.0    (2,5)=48000    (2,9)='(주)이니시스 - ㈜우아한형제들'
(2,14)=48000     (2,15)='복리후생비'
(2,17)='최기환, 문성우, 한원영, 김나연'
(2,18)='야근 식대 4인'
(2,19)=48000     (2,20)=0
```

### 시트 1-1. `1-1. 지출증빙 첨부(쇼핑몰,편의점,마트,문구점 등)`

#### 레이아웃: 좌·우 2페이지, 각 페이지 2×2 그리드 = 총 8개 셀

```
   col 0─3   col 4─7   col 8─11  col 12─15
  ┌────────┬────────┬────────┬────────┐
  │  헤더 좌측 페이지  │  헤더 우측 페이지  │  ← row 0~1 (merged)
  ├────────┼────────┼────────┼────────┤
  │ 셀 1   │ 셀 2   │ 셀 5   │ 셀 6   │  ← row 2~20
  │        │        │        │        │
  ├────────┼────────┼────────┼────────┤
  │ 셀 3   │ 셀 4   │ 셀 7   │ 셀 8   │  ← row 21~39
  │        │        │        │        │
  ├────────┴────────┼────────┴────────┤
  │  좌측 푸터       │  우측 푸터       │  ← row 40~41 (merged)
  └─────────────────┴─────────────────┘
```

#### Merged 셀 좌표 (xlrd 기준 `(rowFirst, rowLast+1, colFirst, colLast+1)`)

```
(0, 2, 0, 8)     좌측 페이지 헤더
(0, 2, 8, 16)    우측 페이지 헤더
(2, 21, 0, 4)    셀 1
(2, 21, 4, 8)    셀 2
(2, 21, 8, 12)   셀 5
(2, 21, 12, 16)  셀 6
(21, 40, 0, 4)   셀 3
(21, 40, 4, 8)   셀 4
(21, 40, 8, 12)  셀 7
(21, 40, 12, 16) 셀 8
(40, 42, 0, 8)   좌측 푸터
(40, 42, 8, 16)  우측 푸터
```

#### 헤더 셀 텍스트
- (0, 0) = `지출증빙 첨부파일(1)`
- (0, 8) = `지출증빙 첨부파일(2)`

#### 푸터 셀 텍스트
- (40, 0), (40, 8) = `↑   ↑      점선에 맞춰 첨부해주시길 바랍니다.      ↑   ↑`

#### 이미지 배치 규칙
- **한 셀 = 한 날짜의 영수증 모음** (PDF 일반경비 신청방법 7p에 명시)
- 셀 안 이미지 정렬: 위→아래, 좌→우 (한 셀에 여러 장이면 적층)
- 동일 날짜 영수증이 한 셀을 넘치면 다음 셀로 넘기지 않고, 한 셀에 합쳐 적층 (예시: PDF 9p)
- 이미지 형식: PWA가 1600px JPEG로 압축해서 보냄

#### ExcelJS 이미지 anchor 계산 (의사 코드)

```ts
// 셀 N(1~8)의 좌상단·우하단 row/col 매핑
const SLOTS = [
  { id: 1, top: 2,  bottom: 21, left: 0,  right: 4  },
  { id: 2, top: 2,  bottom: 21, left: 4,  right: 8  },
  { id: 3, top: 21, bottom: 40, left: 0,  right: 4  },
  { id: 4, top: 21, bottom: 40, left: 4,  right: 8  },
  { id: 5, top: 2,  bottom: 21, left: 8,  right: 12 },
  { id: 6, top: 2,  bottom: 21, left: 12, right: 16 },
  { id: 7, top: 21, bottom: 40, left: 8,  right: 12 },
  { id: 8, top: 21, bottom: 40, left: 12, right: 16 },
]

// 한 슬롯에 N장의 이미지를 적층
function placeImagesInSlot(ws, slot, images) {
  const slotHeightRows = slot.bottom - slot.top  // 19행
  const perImageHeightRows = slotHeightRows / images.length
  images.forEach((img, idx) => {
    const tlRow = slot.top + perImageHeightRows * idx
    const brRow = slot.top + perImageHeightRows * (idx + 1)
    const id = ws.workbook.addImage({ buffer: img.bytes, extension: 'jpeg' })
    ws.addImage(id, {
      tl: { col: slot.left, row: tlRow },
      br: { col: slot.right, row: brRow },
      editAs: 'oneCell',
    })
  })
}
```

### 시트 2. `2. 후불교통,하이패스이용명세서`

- 1차 출시에서 **다루지 않음** (FE1팀 데이터에 없음)
- 시트는 그대로 비워둔 채 출력

### 시트 `지출증빙 첨부_예시`

- 가이드용. 사용자가 보지 않으면 그만. 그대로 보존만.

## 날짜 처리

명세서 날짜는 카드사 파일 시기별로 `2025.10.13`처럼 점으로 오거나 `2026-01-05`처럼 하이픈으로 올 수 있다.
신청서에는 수동 붙여넣기 포맷과 동일하게 원문 문자열을 그대로 쓰고, 셀 서식은 텍스트(`@`)로 둔다.
날짜 매칭·파일명 월 추론처럼 내부 계산이 필요한 곳에서만 `.`/`-` 양쪽 구분자를 모두 허용한다.

## 카드번호·사업자번호·승인번호 처리

- **카드번호**: 명세서에서 이미 마스킹된 형태(`****-3910-0209-3811`). 그대로 복사.
- **사업자번호**: 명세서는 `220-81-55597` 문자열, 채워진 파일에서는 그대로 유지 가능. 숫자 변환은 의미가 없으니 문자열로 유지.
- **승인번호**: 명세서는 `00572725` (앞 0 보존), 채워진 파일은 `572725.0` 형태로 보임. **앞 0 보존을 위해 문자열로 강제** 필요. ExcelJS에서 cell value를 string으로 명시.

## 합계 셀 처리

- (0, 9), (0, 19), (0, 20)은 ExcelJS에서 **수식 대신 값으로 직접 계산해서 입력**.
  - 이유: 호환성 + ExcelJS 수식 평가 미지원으로 다른 셀에서 참조해도 0으로 보임
- 행별 합계 (col 14)도 마찬가지

## 파일명 규칙

```
(카드)제경비신청서_${month}월_${dept}_${name}.xlsx

예: (카드)제경비신청서_11월_FE1팀_최기환.xlsx
```

월은 1~12 정수 (01월 X). 부서·이름은 사용자 입력 그대로.

## 템플릿 변환 (1회 작업)

기존 `.xls` 템플릿을 LibreOffice로 한 번 변환해 앱에 번들:

```
soffice --headless --convert-to xlsx \
  '(카드)제경비신청서_X월_부서_성명.xls'
mv '(카드)제경비신청서_X월_부서_성명.xlsx' \
   apps/web/public/templates/card-expense.xlsx
```

변환 후 손으로 점검할 항목:
- 시트명 4개 보존
- 1-1 시트 merged 셀 8개 보존
- 헤더 텍스트 깨지지 않음
- 행 높이·열 너비(특히 1-1 시트의 슬롯 크기) 보존
