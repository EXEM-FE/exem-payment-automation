## 💻 PC 정산 콘솔 (단일 페이지 퍼널)

상단 진행바(점 4개) sticky. 한 단계씩만 노출, 슬라이드인 애니메이션.

### Step 1 — 모바일에서 데이터 받기
```
모바일에서 데이터 받기
모바일에서 발급받은 PIN 4자리를 입력하면…
사용자 [FE1팀 · 최기환]
PIN    [_][_][_][_]   ← input-otp 4셀
ℹ 모바일 저널에서 "PC로 보내기"를 누르면 PIN이 발급돼요.
[          가져오기          ]
```
PIN 다 입력하면 onComplete 자동 트리거 → 다음 step.

### Step 2 — 명세서 업로드
```
← 이전
명세서를 올려주세요
현대카드 MY COMPANY에서 받은 .xls를…
[ 📂  파일 선택 / 끌어다 놓기 ]
✨ 자동 감지: 이용일자/카드번호/이용자명/이용금액/가맹점/승인번호 등
```
파싱 성공 시 자동으로 다음 step.

### Step 3 — 매칭 검토 (인라인 편집 표)

명세서 = SSoT. 명세서 1행 = 표 1행 = entry 1개 (자동 매칭 또는 자동 시드).
명세서 행은 자동/사용자 어느 쪽으로도 절대 삭제되지 않는다.

```
← 이전
매칭 결과를 확인해주세요
명세서 13건이 모두 표에 있어요 · 자동 11 · 검토 1 · 한도 안내 1

┌─┬────┬───────────────┬───────┬───────────┬───────────┬──────────────┬───────┬──────┬──────┬─┐
│ │일자 │가맹점            │청구    │계정         │함께한 사람  │업무             │신청    │개인   │영수증 │ │
├─┼────┼───────────────┼───────┼───────────┼───────────┼──────────────┼───────┼──────┼──────┼─┤
│●│4/17│이니시스-우아한형제들│48,000│[복리후생비▼]│강 배 김 최  │야근 식대 4인     │48,000│0     │📎 2   │⋯│
│⚠│4/20│소망김밥           │30,900│[복리후생비▼]│최          │야근 식대 1인     │30,900│0     │📎 1   │⋯│
│!│4/21│배달의민족 - 김밥집  │14,500│[복리후생비▼]│최          │야근 식대 1인     │14,500│0     │! 영수증│⋯│
│●│4/21│카카오T            │8,400 │[여비교통비▼]│ -          │[입력]            │8,400 │0     │ 추가  │⋯│
│●│4/22│SK에너지           │64,000│[여비교통비▼]│ -          │[입력]            │64,000│0     │ 추가  │⋯│
└─┴────┴───────────────┴───────┴───────────┴───────────┴──────────────┴───────┴──────┴──────┴─┘
                                                          합계  165,800   165,800   0
                                                              [ 다음 → ]
```

상태 컬럼 (좌측 점):
- ● 자동 매칭 (회색) — 날짜·금액·가맹점 일치
- ◐ 검토 (노랑) — 같은 날 다른 금액/가맹점, 사용자 확인 권장
- ⚠ 한도 안내 (노랑) — 야근/휴일 식대 인당 한도 초과 (다운로드 차단 X)
- ! 영수증 누락 (빨강) — 영수증 필수 가맹점인데 사진 0장 (다운로드 차단 X, 안내만)

표 컬럼 구성 (총 20):
- **명세서(SSoT, readonly)**: 일자, 가맹점, 청구, 카드번호, 이용자, 사원번호, 부서, 사용금액, 사업자번호, 승인번호, 할부, 회차
- **사용자 입력(편집 가능)**: 계정 ▼ (14개 enum), 함께한 사람 (9칩 토글), 업무, 신청금액, 개인 (자동 = 청구 - 신청), 영수증

업무 placeholder 자동 생성:
- 식음료 가맹점 또는 식대성 카테고리이면 `야근 식대 N인` (N = 함께한 사람 수)
- 함께한 사람 토글 시 placeholder의 N이 자동 갱신
- description이 비어있거나 "(야근|휴일) 식대 N인" 패턴이면 자동 갱신, 사용자가 직접 수정한 값은 보존

영수증 셀 (배민 등 다중 사진 지원):
- 클릭 → popover로 썸네일·추가·제거. 한 가맹점에 여러 장 업로드 가능
- 모바일에서 push한 사진(IndexedDB)은 자동으로 표시되며, PC에서도 직접 추가 가능
- 영수증 필수 가맹점에 사진 0장이면 빨간 배지로 안내 (차단은 안 함)

⋯ 메뉴:
- "입력 초기화" — 부착된 entry를 자동 시드 상태로 되돌림. 명세서 행 자체는 삭제되지 않음.

푸터:
- 합계 (청구/신청/개인 SUM)
- 명세서에 없는 모바일 저널 N건 — 출력 미포함 안내

### Step 4 — 미리보기 & 다운로드
```
← 이전
다운로드 준비됐어요
[합계: 270,000원 (13건)]
- 자동 매칭 6건
- 검토 후 채택 1건
- 출력 파일: (카드)제경비신청서_11월_FE1팀_최기환.xlsx

⚠ 50만 초과 행 N건 (있을 때만)
✅ 검증 통과 (영수증·동반인 누락 없음)

[      .xlsx 다운로드      ]
```
다운로드 직후 서버 `DELETE /api/me` 호출 → 슬롯 즉시 폐기 → "완료" 화면.

```
[ 새로 시작 ]
```

## 디자인 원칙 (변경 없음)

1. **모바일 우선** — FAB·풀스크린 시트·세로 카드 흐름
2. **PC도 동일 단일 SPA** — 단계별 풀스크린 퍼널
3. **자주 쓰는 액션은 1탭** — 9명 토글 칩, 식음료 4지선다
4. **위험 액션은 명확히** — 데이터 삭제 빨강, 다운로드 차단 명시
5. **숫자는 천 단위 콤마**, 금액 단위 "원"
6. **PIN 잔여 시간은 디지털 타이머**
